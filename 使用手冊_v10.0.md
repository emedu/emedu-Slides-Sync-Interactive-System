# 📘 伊美：簡報同步互動學習系統 v10.0 使用與部署手冊

---

## **✨ 系統簡介 (伊美：簡報同步互動學習系統 v10.0)**

本系統為 **v10.0 架構重構版**，專為 **伊美：簡報同步互動學習系統** 設計，將原本的 Google Forms 依賴轉為 **全自主 Web App** 介面。
核心功能不變，但體驗與穩定性大幅提升：

*   **📱 手機優先介面**：專為學員手機掃碼設計的響應式網頁。
*   **⚡ 即時互動**：不需等待表單跳轉，直接在網頁上作答並獲取回饋。
*   **🛡️ 安全防護**：內建防秒刷機制，避免重複提交灌水。
*   **🎨 品牌化視覺**：統一使用伊美專屬的海軍藍主視覺。

## **🚀 第一部分：系統部署 (IT/管理員)**

此部分只需執行一次，用於建立您專屬的系統環境。

### **1. 建立 Google Apps Script 專案**
1.  前往 [Google Apps Script 首頁](https://script.google.com/)。
2.  點選左上角 **「新專案」**。
3.  點擊標題「未命名的專案」，重新命名為 `伊美：簡報同步互動學習系統`。

### **2. 植入程式碼**
請將本專案資料夾中的檔案內容，依檔名複製到 GAS 編輯器中：

| 檔案類型 | 檔案名稱 | 說明 |
| :--- | :--- | :--- |
| **程式碼 (.gs)** | `Main.gs` | **入口程式** (重要) |
| | `Service_DB.gs` | 資料庫服務 |
| | `Service_Engine.gs` | 評分與邏輯引擎 |
| | `Service_Security.gs` | 安全驗證 |
| | `Service_AI.gs` | AI 模組 |
| **HTML (.html)** | `UI_Portal.html` | 學員主畫面 |
| | `UI_Action.html` | 前端邏輯 (JavaScript) |
| | `UI_Style.html` | 樣式表 (CSS) |
| | `UI_Admin.html` | **[新增]** 後台管理介面 |

> **注意**：HTML 檔案需在編輯器左側列表按下 `+` -> `HTML` 建立。

### **2.5 設定 AI 金鑰 (重要)**
本系統使用 Google Gemini AI 進行分析，需設定 API Key 才能啟用：
1.  前往 [Google AI Studio](https://aistudio.google.com/) 申請 API Key。
2.  回到 Apps Script 編輯器，點擊左側 **「專案設定」** (齒輪圖示)。
3.  滑動到 **「指令碼屬性 (Script Properties)」** 區塊，點擊 **「新增指令碼屬性」**。
4.  屬性名稱填入：`GEMINI_API_KEY`
5.  屬性值填入：您的 API Key (以 `AIza` 開頭的字串)。
6.  點擊 **「儲存」**。

### **3. 初始化主控台**
1.  在編輯器上方工具列，將執行函式切換為 **`_handleInstall`** (或是 `Service_DB` 內的 `installControlSheet`)。
2.  點擊 **「執行」**。
3.  系統會要求核對權限（存取試算表、雲端硬碟），請點選 **「審查權限」 -> 選擇帳號 -> 進階 -> 前往 (不安全) -> 允許**。
4.  執行完畢後，查看下方的 **「執行記錄」**，會顯示一段網址：`✅ 主控台試算表已建立：https://docs.google.com/...`。
5.  請將該試算表網址存至瀏覽器書籤，這就是您的**後台管理中心**。

### **4. 發布為應用程式**
1.  點擊右上角 **「部署」** -> **「新增部署」**。
2.  左側齒輪選擇 **「網頁應用程式」**。
3.  設定如下：
    *   **說明**：v10.0 正式版
    *   **執行身分**：**我 (Me)** (學員將以您的權限執行，較方便)
    *   **存取權限**：**任何人 (Anyone)** (讓學員不需登入 Google 即可作答)
4.  點擊 **「部署」**。
5.  複製 **「網頁應用程式網址 (Web App URL)」**。這是給學員的連結。

## **⚙️ 第二部分：活動設定 (教學組/講師)**

### **1. 準備新活動 (Setting up)**
本系統 v10.0 採用 **「主控台即活動」** 的單一核心設計。若您要開啟一場新的課程活動，請依序執行：

1.  開啟您的 **主控台試算表** (即部署時產生的 Google Sheet)。
2.  **清空舊資料 (若需要)**：
    *   切換到 **「學員資料總表」**。
    *   保留第 1 列標題，**刪除第 2 列以後的所有舊學員測試資料**。這樣新活動開始時，資料才會乾淨。
3.  **確認題目設定**：
    *   切換到 **「系統設定」** 工作表。
    *   此處即為目前的活動題庫。若需修改或建立新題目，請直接編輯此工作表（詳見下方說明）。

> **⚠️ 注意**：本系統不需透過額外的選單建立活動，只要更改「系統設定」工作表的內容，學員端看到的題目就會即時更新。

### **2. 設計題目**
### **2. 設計題目 (詳細說明)**
請開啟主控台的 **「系統設定」** 工作表，每一列代表一個題目。請依序填入以下欄位：

| 欄位名稱 | 說明 | 填寫範例 |
| :--- | :--- | :--- |
| **階段標籤 (Label)** | **(必填)** 用於區分課程階段，相同標籤的題目會被歸類在同一頁。 | `階段一`、`課後評量` |
| **表單標題 (Form Title)** | 該階段顯示在頁面頂端的標題。 | `課程暖身活動` |
| **表單問題 (Question)** | **(必填)** 學員看到的題目內容。 | `簡報的核心目的是？` |
| **答案欄位名稱** | **(必填)** 資料庫存檔用的欄位 ID (英文/數字)，**不可重複**。 | `Q1_Purpose` |
| **題目類型** | **(必填)** 支援：`單選題`、`多選題`、`簡答題`。 | `單選題` |
| **選項** | 若為選擇題，請用半形逗號 `,` 分隔選項。 | `傳遞價值,展示數據,填滿時間` |
| **標準答案** | 用於自動評分。選擇題請填選項文字；多選用 `,` 分隔。 | `傳遞價值` |
| **分數** | 該題答對獲得的分數。 | `10` |
| **提示文字** | (選填) 顯示在題目下方的補充說明。 | `請選擇最合適的一個答案` |

> **💡 小技巧**：
> *   **「階段標籤」** 決定了題目出現的順序與分組。當學員完成一個階段的所有題目後，系統才會解鎖下一個階段（若有設定）。
> *   **「答案欄位名稱」** 請勿使用特殊符號或中文，建議使用 `Q1`, `Q2` 簡單編號即可。

### **3. 產生 QR Code (詳細步驟)**

**步驟 A：取得學員連結**
1.  回到 Apps Script 編輯器。
2.  點擊右上角 **「部署」** -> **「管理部署」**。
3.  點擊左側「使用中 (Active)」的項目。
4.  複製 **「網頁應用程式網址 (Web App URL)」** (以 `https://script.google.com/...` 開頭)。
    *   *注意：請確認權限設定為「任何人 (Anyone)」，否則學員無法進入。*

**步驟 B：製作 QR Code**
您可以選擇以下任一方式將網址轉為 QR Code：
*   **方法一 (Chrome 瀏覽器)**：
    1.  在這個網址按右鍵（或點擊網址列右側的分享圖示）。
    2.  選擇 **「建立 QR 圖碼」**。
    3.  下載圖片並貼到您的投影片首頁。
*   **方法二 (線上產生器)**：
    1.  搜尋「QR Code 產生器」。
    2.  貼上您的 Web App URL。
    3.  下載圖片。

> **💡 進階用法**：
> 若您希望網址短一點，可配合 **Bit.ly** 或 **Reurl** 縮網址服務，再製作 QR Code，方便現場學員手動輸入。

## **📱 第三部分：學員操作 (學生)**

### **1. 進入系統**
*   學員掃描 QR Code 或點擊連結。
*   看到 **「伊美：簡報同步互動學習系統」** 還有 **海軍藍** 背景的介面。

### **2. 登入**
*   首次需輸入 **學號** (必填)。
*   系統會自動檢查該學號的歷史進度。

### **3. 互動與作答**
*   介面會顯示目前的題目。
*   學員輸入答案或選擇選項。
*   點擊 **「提交答案」**。

### **4. 即時回饋與 AI 分析**
*   **AI 評語**：針對簡答題，系統會呼叫 Gemini AI 進行分析，提供針對性的評語與建議。
*   **正確答案**：選擇題提交後，若有設定標準答案，系統會顯示正確與否。
*   **進度追蹤**：上方進度條會顯示目前階段的完成度。

## **🔧 常見問題 (FAQ)**

**Q: 如何修改題目？**
A: 直接在主控台試算表修改，下次學員載入頁面時（視快取策略）會抓取最新設定。建議建立新活動來區隔不同梯次。

**Q: 學生反映無法提交？**
A: v10.0 有防秒刷機制，請確認學生是否在 60 秒內重複按送出。若遇網路問題，請重新整理頁面。

**Q: 資料存在哪裡？**
A: 所有的作答紀錄都會寫入與活動綁定的 **「學員資料總表」** (Google Sheet)，您可以隨時匯出 Excel。

## **👨‍💼 第四部分：後台管理 (講師/助教)**

現在您可以直接透過網頁進入後台，不需每次都開啟 Google Drive。

### **1. 登入後台**
*   使用與學員相同的 Web App 網址，但在最後加上 `?route=admin`。
    *   範例：`https://script.google.com/.../exec?route=admin`
*   輸入管理員密碼（預設：`admin123`）。

### **2. 功能總覽**
登入後，您可以看到：
*   **即時數據**：總學員數、今日提交數。
*   **快速入口**：一鍵開啟「主控台試算表」與「學員資料總表」。

> **小技巧**：建議將 `?route=admin` 的網址加入瀏覽器「我的最愛」，方便隨時查看。

### **3. 課程內容編輯 (CMS)**
本系統內建簡易的 CMS 功能，讓您直接在網頁上新增題目：
1.  **選擇階段**：選擇現有階段或輸入新階段名稱。
2.  **填寫題目**：輸入問題內容、選擇題型（單選/簡答）、設定配分與標準答案。
3.  **儲存**：點擊「新增此題」，系統會自動將題目寫入後台試算表的「系統設定」頁籤中。
