## 一、系統架構說明

本系統以 **Google Apps Script (GAS)** 為後端執行環境，採 **MVC 模組化架構**設計。

```
┌─────────────────────────────────────────────────────┐
│                   Google Apps Script                │
│                                                     │
│  ┌──────────┐   ┌──────────────┐  ┌──────────────┐ │
│  │ Main.gs  │──▶│Service_DB.gs │  │Service_AI.gs │ │
│  │ 控制器    │   │ 資料存取層    │  │ AI 整合層     │ │
│  └─────┬────┘   └──────────────┘  └──────────────┘ │
│        │        ┌──────────────┐  ┌──────────────┐ │
│        └───────▶│Service_      │  │Service_      │ │
│                 │Engine.gs     │  │Security.gs   │ │
│                 │ 業務邏輯      │  │ 安全驗證      │ │
│                 └──────────────┘  └──────────────┘ │
└─────────────────────────────────────────────────────┘
         │                    │
         ▼                    ▼
  ┌─────────────┐    ┌──────────────┐
  │ Google      │    │  Gemini API  │
  │ Sheets      │    │ (gemini-2.5) │
  └─────────────┘    └──────────────┘
```

### 模組職責

| 模組 | 職責 |
|-------|------|
| `Main.gs` | Web App 入口，路由請求至 `/portal`（學員）或 `/admin`（管理員） |
| `Service_DB.gs` | 試算表 CRUD、主控台管理、活動設定讀取 |
| `Service_Engine.gs` | 評分邏輯（單選/多選/簡答）、進度追蹤、階段轉換 |
| `Service_Security.gs` | 防秒刷、身份驗證、管理員登入 |
| `Service_AI.gs` | Gemini API 呼叫、結構化 JSON 回饋、Coach Persona |

### 前端檔案

| 檔案 | 用途 |
|-------|------|
| `UI_Portal.html` | 學員互動主頁（QR Code 掃描後進入）|
| `UI_Action.html` | 前端邏輯腳本（表單驗證、非同步提交）|
| `UI_Style.html` | 全域 CSS 樣式、emedu 品牌色系 |
| `UI_Admin.html` | 後台管理系統（登入 + CMS 儀表板）|

## 二、安裝與啟動指南

### 前置條件
- Google 帳號（會自動擁有 Google Sheets 與 GAS 存取權）
- Gemini API Key（從 [Google AI Studio](https://aistudio.google.com/) 取得）

### 部署步驟

1. **建立 GAS 專案**
   - 前往 [script.google.com](https://script.google.com) 新增專案
   - 將所有 `.gs` 與 `.html` 檔案內容分別複製至 GAS 編輯器中對應的檔案

2. **設定 API Key**
   - 在 GAS 編輯器，前往「專案設定」→「指令碼屬性」
   - 新增屬性：`GEMINI_API_KEY` = `你的 API Key`

3. **初始化主控台試算表**
   - 在 GAS 編輯器中選擇 `Main.gs`
   - 執行 `Service_DB.installControlSheet()` 函式
   - 查看執行日誌取得試算表連結

4. **發布為 Web App**
   - 點選「部署」→「新增部署」→「網頁應用程式」
   - 執行身分：**我（Me）**
   - 存取權限：**任何人（Anyone）**
   - 複製取得的 Web App URL

5. **驗證**
   - 在瀏覽器開啟 Web App URL，應看到學員介面（`UI_Portal`）
   - 在 URL 後加上 `?page=admin`，應看到管理員登入頁

## 三、核心 API 說明

### 後端 API（供前端 `google.script.run` 呼叫）

| 函式名稱 | 說明 | 回傳格式 |
|----------|----|----------|
| `apiGetQuestion(stageId)` | 取得指定階段的當前問題 | `{question, type, options}` |
| `apiSubmitAnswer(payload)` | 提交學員答題資料 | `{success, score, feedback}` |
| `apiAdminLogin(password)` | 管理員登入驗證 | `{success, token}` |
| `apiGetAdminData(token)` | 取得後台儀表板資料 | `{stages, responses, stats}` |
| `apiGetStageList()` | 動態取得所有課程階段列表 | `[{id, name}]` |
| `apiAddQuestion(data)` | 新增題目（含新建階段支援）| `{success, questionId}` |
| `apiGetAIFeedback(context)` | 呼叫 Gemini 取得 AI 回饋 | `{feedback, suggestions, score}` |

### AI 回傳 JSON 結構（`Service_AI.gs`）

```json
{
  "feedback": "做得很好！你掌握了核心概念。",
  "suggestions": ["試著思考更多應用場景", "複習第三章的定義"],
  "encouragement": "你的進步非常明顯，繼續加油！",
  "score_analysis": "答案涵蓋了 80% 的關鍵要素"
}
```

## 四、評分規則參考

| 題型 | 規則 |
|------|------|
| 單選題 | 完全比對，答對得滿分 |
| 多選題 | 部分給分；多選錯項倒扣 |
| 簡答題 | 模糊比對（關鍵字命中率計分）|
