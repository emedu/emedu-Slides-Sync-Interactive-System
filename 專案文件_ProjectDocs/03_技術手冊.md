## 一、系統架構說明

本系統以 **Google Apps Script (GAS)** 為後端執行環境，採 **MVC 模組化架構**設計。

```
┌─────────────────────────────────────────────────────┐
│                   Google Apps Script                │
│                                                     │
│  ┌──────────┐   ┌──────────────┐  ┌──────────────┐ │
│  │ Main.gs  │──▶│Service_DB.gs │  │Service_AI.gs │ │
│  │ 控制器    │   │ 資料存取層    │  │ AI 整合層     │ │
│  └─────┬────┘   └──────────────┘  └──────────────┘ │
│        │        ┌──────────────┐  ┌──────────────┐ │
│        └───────▶│Service_      │  │Service_      │ │
│                 │Engine.gs     │  │Security.gs   │ │
│                 │ 業務邏輯      │  │ 安全驗證      │ │
│                 └──────────────┘  └──────────────┘ │
└─────────────────────────────────────────────────────┘
         │                    │
         ▼                    ▼
  ┌─────────────┐    ┌──────────────┐
  │ Google      │    │  Gemini API  │
  │ Sheets      │    │  (Gemini 3)  │
  └─────────────┘    └──────────────┘
```

### 模組職責

| 模組 | 職責 |
|-------|------|
| `Main.gs` | Web App 入口，路由請求至 `/portal`（學員）或 `/admin`（管理員） |
| `Service_DB.gs` | 試算表 CRUD、主控台管理、活動設定讀取 |
| `Service_Engine.gs` | 評分邏輯（單選/多選/簡答）、進度追蹤、階段轉換 |
| `Service_Security.gs` | 防秒刷、身份驗證、管理員登入 |
| `Service_AI.gs` | Gemini 3 API 呼叫、結構化 JSON、錯誤透傳機制 |

### 前端檔案

| 檔案 | 用途 |
|-------|------|
| `UI_Portal.html` | 學員互動主頁（QR Code 掃描後進入）|
| `UI_Action.html` | 前端邏輯腳本（表單驗證、非同步提交）|
| `UI_Style.html` | 全域 CSS 樣式、emedu 品牌色系 |
| `UI_Admin.html` | 後台管理系統（登入 + CMS 儀表板）|

## 二、安裝與啟動指南

### 前置條件
- Google 帳號（會自動擁有 Google Sheets 與 GAS 存取權）
- Gemini API Key（從 [Google AI Studio](https://aistudio.google.com/) 取得）

### 部署步驟

1. **建立 GAS 專案**
   - 前往 [script.google.com](https://script.google.com) 新增專案
   - 將所有 `.gs` 與 `.html` 檔案內容分別複製至 GAS 編輯器中對應的檔案

2. **設定 API Key**
   - 在 GAS 編輯器，前往「專案設定」→「指令碼屬性」
   - 新增屬性：`GEMINI_API_KEY` = `你的 API Key`

3. **初始化主控台試算表**
   - 在 GAS 編輯器中選擇 `Main.gs`
   - 執行 `Service_DB.installControlSheet()` 函式
   - 查看執行日誌取得試算表連結

4. **發布為 Web App**
   - 點選「部署」→「新增部署」→「網頁應用程式」
   - 執行身分：**我（Me）**
   - 存取權限：**任何人（Anyone）**
   - 複製取得的 Web App URL

5. **驗證**
   - 在瀏覽器開啟 Web App URL，應看到學員介面（`UI_Portal`）
   - 在 URL 後加上 `?page=admin`，應看到管理員登入頁

## 三、核心 API 說明

### 後端 API（供前端 `google.script.run` 呼叫）

| `apiGetAppUrls()` | [v10.2.3.1] 自動偵測載入入口網址 | `{portal, admin, status}` |
| `apiGetActivityList()` | [v10.3.0] 取得所有活動清單 | `{activities: []}` |
| `apiCreateActivity(name)` | [v10.3.0] 建立新活動（三分頁） | `{status, activityId}` |
| `apiSwitchActivity(id)` | [v10.3.0] 切換當前活躍活動 | `{status}` |
| `apiGetQuestion(stageId)` | 取得指定階段的當前問題 | `{question, type, options}` |
| `apiSubmitAnswer(payload)` | 提交學員答題資料 | `{success, score, feedback}` |
| `apiAdminLogin(password)` | 管理員登入驗證 | `{success, token}` |
| `apiGetAdminData(token)` | 取得後台儀表板資料 | `{stages, responses, stats}` |
| `apiGetStageList()` | 動態取得所有課程階段列表 | `[{id, name}]` |
| `apiAddQuestion(data)` | 新增題目（含新建階段支援）| `{success, questionId}` |
| `apiLogin(studentId)` | 登入並取得當前階段所有題目清單 | `{tasks: [], userIndex: 0}` |

## 四、多活動路由架構 (v10.3.0)

本版本導入了「動態路由」機制，各業務邏輯不再硬編碼分頁名稱。

### 1. 核心控制器：Script Properties
系統透過 `ACTIVE_ACTIVITY_ID` 決定當前指向哪一個活動。
- **空值**：指向預設分頁（`系統設定`、`學員資料總表`）。
- **有值**：指向自訂分頁（`[ID]-設定`、`[ID]-學員`）。

### 2. 資料層路由模式 (Service_DB.gs)
```javascript
getActiveDataSheetName: function() {
  const activeId = this.getActiveActivityId();
  if (!activeId) return '學員資料總表';
  return '[' + activeId + ']-學員';
}
```

### 3. 分頁自動建置流程
當呼叫 `createActivity` 時，系統自動執行：
1. 複製標題列與格式，建立 `[名]-設定`、`[名]-學員`、`[名]-追蹤`。
2. 在 `活動清單` 中註冊新 ID。

### AI 回傳 JSON 結構（`Service_AI.gs`）

```json
{
  "rating": "良好",
  "feedback": "做得很好！你掌握了核心概念。",
  "suggestions": ["試著思考更多應用場景", "複習第三章的定義"]
}
```
*註：v10.3.8 加入了 rating 欄位以支援星星評分顯示。*

## 五、GAS 特定環境與開發限制 (備忘)

1. **Modal Dialog 渲染沙盒限制 (卡關紀錄)**：
   - 當使用 `SpreadsheetApp.getUi().showModalDialog()` 呈現 HTML 時，前端特定的事件監聽（如 `window.onload` 或 `DOMContentLoaded`）可能會受到沙盒限制而不觸發。
   - 解決方針應改採 Inline Script (在 `<body>` 底部直接呼叫初始化函式) 確保 JS 順利載入。
