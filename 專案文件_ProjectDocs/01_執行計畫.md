# 01\_執行計畫 — emedu-Slides-Sync-Interactive-System

> **專案名稱**：emedu-Slides-Sync-Interactive-System（前身：伊美：簡報同步互動學習系統）
> **協定版本**：Antigravity 跨裝置工作區同步協定 v3.0
> **最後更新**：2026-03-04

---

## 一、專案目標

打造一套「簡報同步 × 即時互動」的線上教學輔助系統，讓講師透過 Google Slides 中的 QR Code，以零延遲的方式收集學員答題資料，並透過 Gemini AI 給予個人化學習回饋。

## 二、開發步驟規劃

### 階段 0：需求分析與規格確認 ✅
- 分析 v9.4.2 單一腳本架構的限制
- 制訂 v10.0 技術開發白皮書（MVC 模組化架構）
- 確認品牌名稱更新為 **emedu-Slides-Sync-Interactive-System**

### 階段 1：核心後端重構 ✅
- 拆分五個 `.gs` 服務模組（Main / DB / Engine / Security / AI）
- 遷移 v9.4.2 所有業務規則（評分邏輯、進度追蹤）

### 階段 2：前端介面重構 ✅
- 實作響應式行動優先學員端（UI_Portal）
- 實作前端邏輯腳本（UI_Action）

### 階段 4：Admin CMS 動態載入增強 ✅（2026-03-03）

### 階段 5：AI / NLP 能力提升 ✅（2026-03-03）

- 強化 Prompt Engineering（導入教練人格 Coach Persona）
- 實作上下文感知回饋（Context-Aware Feedback）

### 階段 6：系統穩定化與自動化部署 (v10.2) ✅（2026-03-04）
- 實作網頁版安裝精靈 (`UI_Setup`)，解決初始化悖論
- 補齊自動化觸發器部署 (`setupInstallableTriggers`)
- 實作「智慧管理控制台」(`UI_Console`) 與雙層網址快取機制
- 深度邏輯修補：修復無資料時的 Null 崩潰、API `getUrl()` 掛起等邊際案例

### 階段 6.1：主控台介面通訊掛起深度除錯（卡關中，進行中）
- **現象**：主控台 UI (`UI_Console.html`) 版面已更新並顯示使用操作指引，但原有的 JS 逾時判斷與 `google.script.run` 呼叫皆完全未觸發。網址持續停在「正在獲取網址...」。
- **問題分析**：在 Google Spreadsheet Modal dialog（模態框）環境中，`window.onload` 與 `DOMContentLoaded` 事件遭攔截或提前失效。
- **後續計畫**：拔除事件綁定，改用 HTML body 結尾強制同步執行腳本（Inline execution），並設置更強韌的錯誤回報機制（前端 DOM 直接顯示 Console.error 輸出）。

### 階段 8：多活動並行管理 (v10.3.0) ✅（2026-03-04）
- 實作「多活動」架構，單一系統可管理多個課程/班級
- 透過 Script Properties 控制當前活躍活動 ID
- 實現分頁動態導航（`[活動名]-設定`、`[活動名]-學員`）
- 重構使用說明介面為卡片式 UI，並擴充多活動操作手冊
