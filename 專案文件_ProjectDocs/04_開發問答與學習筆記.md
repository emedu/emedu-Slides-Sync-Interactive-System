## 第一章：架構設計

**Q1：為什麼要從 v9.4.2 的單一腳本重構為 v10.0 的 MVC 架構？**

A：v9.4.2 的單一腳本雖然功能完整，但隨著功能增加，維護困難度指數成長。MVC 拆分能達到：
- **可測試性**：各服務模組可獨立單元測試
- **可維護性**：修改 DB 邏輯不影響評分引擎
- **可擴展性**：新增 AI 服務只需加一個 `Service_AI.gs`

---

**Q2：為什麼選擇 Google Sheets 作為資料庫，而非真正的資料庫？**

A：因為目標用戶（教師/教育機構）已有 Google Workspace 帳號，且 GAS ↔ Sheets 整合無需額外設定或費用。對於每次最多數十到百位學員的互動量，Sheets 效能完全足夠。

---

## 第二章：AI 整合

**Q3：Gemini API 呼叫時出現「models/gemini-3.0-flash not found」錯誤，如何處理？**

A：Gemini 3.0 尚未正式發布，API 不存在此模型。解決方案是回退至 `gemini-2.5-flash`（已確認可用的穩定版本）。
```javascript
// ❌ 錯誤
const model = 'gemini-3.0-flash';

// ✅ 正確
const model = 'gemini-2.5-flash';
```

---

**Q4：如何讓 Gemini AI 回傳結構化 JSON，而非自由文字？**

A：在 Prompt 中明確要求 JSON 格式，並指定欄位。範例 Prompt 結尾：
```
請以下列 JSON 格式回答，不要有任何額外文字：
{
  "feedback": "...",
  "suggestions": ["...", "..."],
  "encouragement": "...",
  "score_analysis": "..."
}
```
若 AI 回傳的字串包含 markdown 代碼塊（```json ... ```），需在程式中先清除再 `JSON.parse()`。

---

**Q5：什麼是「Coach Persona（教練人格）」？為什麼要導入？**

A：傳統 AI 回饋易流於「你的答案是對的/錯的」，缺乏激勵感。導入教練人格後，Prompt 加入指示：
- 以正向、鼓勵的語氣回饋
- 指出優點再提改進方向
- 避免直接說「錯了」，改為「試著想想…」

結果：學員實際互動體驗顯著提升。

## 第三章：Admin CMS

**Q6：為什麼課程階段列表要改為動態載入？**

A：原先 `UI_Admin.html` 的「新增題目」表單中，課程階段是硬編碼的靜態清單。若管理員透過試算表直接新增階段，CMS 畫面不會自動更新，造成資料不同步。

解決方案：新增 `apiGetStageList()` API，前端在表單開啟時呼叫此 API 取得最新列表。

---

**Q7：如何處理「新增題目時，若指定階段不存在」的情境？**

A：在 `apiAddQuestion()` 的處理邏輯中，先呼叫 `Service_DB.getStageById()` 確認階段存在。若不存在，自動呼叫 `Service_DB.createStage()` 建立新階段後，再寫入題目。前端只需在表單加一個「這是新階段」的勾選框。

---

## 第四章：安全與防護

**Q8：「防秒刷」機制是如何實作的？**

A：`Service_Security.gs` 中的 `withSubmitGuard_()` 函式：
1. 記錄每位學員最後一次提交的時間戳記至 Sheets
2. 每次提交時，比對當前時間與上次提交時間
3. 若間隔 < `SUBMIT_GUARD_SECONDS`（預設 10 秒），直接拒絕並回傳錯誤

---

## 第五章：部署常見問題

**Q9：部署後訪問 Web App 出現「You need permission」，如何解決？**

A：發布時「存取權限」必須設為「任何人（Anyone）」，而非「只有自己」。確認後重新部署即可。

---

**Q10：Script Properties 如何設定？**

A：在 GAS 編輯器：
1. 左側選單 → 「專案設定（齒輪圖示）」
2. 下滑至「指令碼屬性」→ 「新增屬性」
3. 屬性名稱：`GEMINI_API_KEY`，值：你的 API Key
4. 在程式中用 `PropertiesService.getScriptProperties().getProperty('GEMINI_API_KEY')` 讀取

## 第七章：自動化部署工具與環境排除 (clasp & Terminal)

**Q12：什麼是 clasp？為什麼要使用它？**

A：`clasp` (Command Line Apps Script Projects) 是 Google 官方提供的指令碼工具。
- **優點**：讓我們能在本地（如 VS Code）撰寫代碼，並像使用 Git 一樣透過指令同步到 Google 雲端。
- **核心價值**：避免手動複製貼上，支持本地開發環境的所有優點（如 AI 輔助、程式碼提示）。

---

**Q13：今日學習到的常用 clasp 指令有哪些？**

A：
1. `clasp login`：啟動 Google 帳號授權流程。
2. `clasp create --title "專案名稱" --type webapp`：在雲端建立新的指令碼專案。
3. `clasp push -f`：將本地所有檔案強制推送到雲端（會覆蓋雲端原有內容）。
4. `clasp open`：在瀏覽器中直接開啟當前專案的 GAS 編輯器網頁。

---

**Q14：執行 clasp 時出現「ReferenceError: require is not defined」是為什麼？**

A：這是因為本地的測試腳本（如 `local_test_runner.js`）使用了 Node.js 的語法（`require`），而這些檔案被同步到了不支援 Node.js 的 Google Apps Script 環境。
**解決方案**：建立 `.claspignore` 檔案，將所有非 `.gs` 或 `.html` 的本地檔案排除在同步範圍外。

---

**Q15：今日學習到的終端機 (Terminal) 基礎操作？**

A：
- **開啟終端機**：快捷鍵 `Ctrl + \`` 或選單「檢視 -> 終端機」。
- **路徑切換 (Windows)**：使用 `cd "路徑名稱"` 進入資料夾。若路徑包含空格或特殊符號（如「」），務必使用雙引號括起來。
- **分頁管理**：終端機視窗右側可切換不同進程（如編譯中、開發中、或是登入授權中）的分頁。

---

**Q16：如果 clasp 推送失敗，提示「User has not enabled the Apps Script API」，該怎麼辦？**

A：這代表您的 Google 帳號尚未開啟開發者權限。需前往 [script.google.com/home/usersettings](https://script.google.com/home/usersettings) 將「Google Apps Script API」切換為 **開啟 (On)**。
